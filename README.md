# Large-Scale Data Analysis with MapReduce

This project analyzes the **Uber NYC July 2014 trip dataset** using a MapReduce pattern.

It supports two execution modes:
- **Local Python pipeline** (`main.py`) for quick development and validation.
- **Hadoop Streaming in WSL** (`run_hadoop.sh`) for distributed-style processing.

The core task is to compute **hourly trip counts** (00 to 23), then generate reports and visualizations.

## Project Goals

- Demonstrate a clean MapReduce workflow using Python mapper/reducer scripts.
- Show both local and Hadoop Streaming execution approaches.
- Produce machine-readable outputs (`.json`) and human-readable summaries (`.txt`).
- Generate visual charts for hourly and daily-average analysis.

## Dataset Information

- File: `data/uber-raw-data-jul14.csv`
- Period: July 2014
- Domain: Uber pickups in New York City
- Columns:
1. `Date/Time` (pickup timestamp)
2. `Lat` (latitude)
3. `Lon` (longitude)
4. `Base` (dispatch base code)

Example rows:

```csv
"Date/Time","Lat","Lon","Base"
"7/1/2014 0:03:00",40.7586,-73.9706,"B02512"
"7/1/2014 0:05:00",40.7605,-73.9994,"B02512"
```

## How the Pipeline Works

MapReduce flow:

1. **Mapper (`mapper.py`)**
   - Reads CSV lines from `stdin`.
   - Parses hour from `Date/Time`.
   - Emits `HH<TAB>1`.

2. **Sort phase**
   - Local mode performs explicit sort in `main.py` before reducer.
   - Hadoop Streaming performs shuffle/sort automatically.

3. **Reducer (`reducer.py`)**
   - Aggregates grouped keys by hour.
   - Emits `Hour HH:00<TAB>count`.

4. **Post-processing**
   - Local mode writes `output/hourly_trips.json`.
   - Hadoop mode writes `results/output.txt` and `results/output_daily_avg.json`.
   - `visualize.py` creates charts and short statistics reports.

## Prerequisites

### Local Python Mode

- Python 3.9+
- `pip`
- Python packages:
  - `matplotlib`
  - `numpy`

### Hadoop Mode (WSL)

- WSL (Ubuntu recommended)
- Java installed
- Hadoop installed and configured
- Hadoop daemons available (`NameNode`, `DataNode`, `ResourceManager`, `NodeManager`)
- Environment variable `HADOOP_HOME` set
- Hadoop Streaming JAR discoverable under `HADOOP_HOME`

## Run Guide: Local Python Pipeline

From project root:

```bash
# Optional but recommended
python -m venv .venv

# Windows PowerShell
.\.venv\Scripts\Activate.ps1

# WSL / Linux
source .venv/bin/activate

pip install matplotlib numpy
python main.py
python visualize.py
```

What this produces:
- `output/hourly_trips.json`
- `output/hourly_report.txt`
- `output/hourly_bar_chart.png`

## Run Guide: Hadoop Streaming in WSL

Open WSL terminal at project root and run:

```bash
chmod +x run_hadoop.sh mapper.py reducer.py
export HADOOP_HOME=/path/to/hadoop
./run_hadoop.sh
```

Optional custom input file:

```bash
./run_hadoop.sh data/uber-raw-data-jul14.csv
```

Then generate visuals for Hadoop outputs:

```bash
python3 visualize.py
```

What `run_hadoop.sh` does automatically:

1. Validates required files and Hadoop variables.
2. Starts Hadoop services if needed.
3. Uploads CSV to HDFS input path.
4. Runs Hadoop Streaming job with `mapper.py` and `reducer.py`.
5. Downloads reducer output to `results/output.txt`.
6. Creates `results/output_daily_avg.json` using 31-day divisor.

## Output Format Details

### Local JSON (`output/hourly_trips.json`)

Contains:
- `timestamp`
- `data` object with keys `"00"` to `"23"`
- `summary` with total trips, hour count, and average

Example:

```json
{
  "data": {
    "00": 17953,
    "01": 11527
  },
  "summary": {
    "total": 796121,
    "hours": 24,
    "average": 33171.708333333336
  }
}
```

### Hadoop Text Output (`results/output.txt`)

Format:

```text
Hour 00:00	17953
Hour 01:00	11527
...
```

### Hadoop Daily Average JSON (`results/output_daily_avg.json`)

Contains monthly totals converted to **daily average per hour**:

```json
{
  "metric": "daily_average_per_hour",
  "days_divisor": 31,
  "data": {
    "17": 1879.35
  },
  "summary": {
    "total_monthly_trips": 796121,
    "average_daily_trips": 25681.32
  }
}
```

## Troubleshooting

- `source .venv/bin/activate` fails in PowerShell:
  - Use `.\.venv\Scripts\Activate.ps1` in PowerShell.

- Hadoop script fails with `HADOOP_HOME is not set`:
  - Export the variable before running script.

- Hadoop script cannot find streaming JAR:
  - Verify `$HADOOP_HOME` and ensure `hadoop-streaming*.jar` exists.

- Reducer results look incorrect in custom local runs:
  - Ensure mapper output is sorted by key before reducer input.
